                              PROBLEMA DETECTADO
═══════════════════════════════════════════════════════════════════════════════

ALERTA DEL NAVEGADOR:
"Para mostrar esta página, Firefox necesita enviar información que repetirá
cualquier acción (como una búsqueda o una confirmación de compra) realizada
anteriormente."

CAUSA: Después de procesar un POST, la página se recargaba sin redirigir,
       dejando al usuario en una página que contiene un formulario que ya
       fue procesado.

FLUJO PROBLEMÁTICO:
──────────────────

1. Usuario envía formulario (POST)
   └─ /admin/items/index.php (POST)
   
2. Servidor procesa el POST y mostró la página
   └─ Misma URL /admin/items/index.php (GET)
   
3. Firefox detecta que la URL tiene historial POST
   └─ Advierte que el formulario ya fue procesado
   
4. Si el usuario actualiza, Firefox pregunta:
   └─ "¿Deseas reenviar el formulario?"

═══════════════════════════════════════════════════════════════════════════════
                    SOLUCIÓN: PATRÓN PRG (POST-REDIRECT-GET)
═══════════════════════════════════════════════════════════════════════════════

El patrón PRG es una arquitectura estándar que elimina este problema:

FLUJO CORRECTO:
───────────────

1. Usuario envía formulario (POST)
   └─ /admin/items/index.php (POST)
   
2. Servidor PROCESA el POST
   └─ Realiza la acción (create, update, delete)
   └─ Almacena mensaje en sesión
   
3. Servidor REDIRIGE con GET
   └─ header('Location: /admin/items/index.php')
   
4. Navegador CARGA la página con GET
   └─ Sin historial POST → Sin advertencias
   
5. Servidor MUESTRA el mensaje guardado en sesión
   └─ Usuario ve el resultado
   └─ URL es segura para actualizar

PASO 1: Procesar el POST
────────────────────────

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $action = $_POST['action'] ?? '';
    $redirect = false;  // Bandera para activar redirección
    
    if ($action === 'create') {
        if ($itemClass->create($data)) {
            $_SESSION['success'] = 'Item creado';
            $redirect = true;  ← ÉXITO: Marcar redirección
        } else {
            $error = 'Error al crear';  ← FALLO: Mantener en página
        }
    }
}

PASO 2: Redirigir si es exitoso
────────────────────────────────

if ($redirect) {
    header('Location: ' . SITE_URL . 'admin/items/index.php');
    exit;  ← IMPORTANTE: exit/die para detener ejecución
}

PASO 3: Mostrar el mensaje guardado
────────────────────────────────────

// Ya estamos en una petición GET nueva
if (isset($_SESSION['success'])) {
    $success = $_SESSION['success'];        ← Obtener mensaje
    unset($_SESSION['success']);            ← Limpiar para siguiente carga
}

// En el HTML
<?php if ($success): ?>
    <div class="alert alert-success">
        <?php echo htmlspecialchars($success); ?>
    </div>
<?php endif; ?>

5. Servidor guarda mensaje en sesión
6. Servidor REDIRIGE a /admin/usuarios/index.php (GET)
7. Navegador carga página con GET
8. Servidor muestra mensaje de éxito
9. Usuario actualiza página de forma segura (GET)

CASO 2: CREAR USUARIO CON FALLO
────────────────────────────────

1. Usuario envía formulario con datos incompletos
2. Envía POST
3. Servidor valida datos ❌
4. Servidor asigna error a variable local $error
5. $redirect permanece false
6. Servidor NO redirige
7. Servidor muestra formulario + mensaje de error
8. Usuario no ve advertencia porque el POST no fue exitoso
9. Usuario puede corregir y reintentar
