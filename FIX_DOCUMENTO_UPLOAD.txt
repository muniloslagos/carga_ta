# ✅ ERRORES CORREGIDOS - CARGA DE DOCUMENTOS

## Problemas Encontrados

### 1. **Error: "Column 'usuario_id' cannot be null"**

**Causa**: En `enviar_documento.php` se buscaba `$_SESSION['usuario_id']` pero la sesión usa `$_SESSION['user_id']`

**Impacto**: Cuando un usuario intentaba cargar un documento, el sistema no podía obtener su ID, resultando en NULL

**Solución**: 
- Cambié `$_SESSION['usuario_id']` por `$_SESSION['user_id']` (línea 13)
- Agregué validación adicional para verificar que user_id existe (línea 30-35)
- Mejor manejo de errores con mensajes claros

### 2. **Warning: Array to string conversion**

**Causa**: El archivo se pasaba como array `$_FILES['archivo']` directo a la BD

**Impacto**: PHP generaba un warning sobre conversión de tipos

**Solución**: 
- El flujo ya procesaba el archivo correctamente en `Documento::create()`
- Se agregó mejor validación de entrada

### 3. **Modal sin información de período**

**Causa**: El modal no mostraba qué mes/período estaba cargando

**Impacto**: Usuario confundido sin saber para qué mes estaba subiendo

**Solución**:
- Actualicé el modal para mostrar:
  - "Está cargando el item: [Nombre]"
  - "para el mes de [Mes Año]"
- Mejoré la función JavaScript `seleccionarItem()` para mostrar el período
